# 開発チーム構成 - 企業セキュリティ診断ツール

このプロジェクトでは以下のエージェントチームで開発を進める。
タスクを受け取ったら、PMが分解し、各エージェントに並列で委譲すること。

---

## チーム構成

### PM（プロジェクトマネージャー）
- 役割：要件の整理・タスク分解・各エージェントへの指示・進捗管理
- 作業開始時に必ずタスクをTodoWriteで整理してから各エージェントに委譲する
- 完了後にコード全体の整合性を確認する

### フロントエンドエージェント
- 役割：HTML/CSS/JavaScript・UI設計・UX改善
- 対象ファイル：`index.php`, `assessment.php`, `result.php`, `css/style.css`
- 方針：シンプルで営業資料として使えるプロフェッショナルなデザイン

### バックエンドエージェント
- 役割：PHPロジック・リスク計算式・データ処理・セッション管理
- 対象ファイル：`includes/calculator.php`, `result.php`（ロジック部分）
- 方針：計算根拠を明確にし、業界統計に基づいた信頼性の高い算出を行う

### セキュリティレビューエージェント
- 役割：コードの脆弱性チェック・XSS/SQLインジェクション対策・入力値検証
- すべての実装完了後に必ずレビューを実施する
- 問題を発見したら即座に修正指示を出す

### QAエージェント
- 役割：動作確認・バグ検出・エッジケーステスト・出力値の妥当性確認
- 実装・セキュリティレビュー後に実施する
- テスト観点をリストアップして確認する

---

## 開発フロー

```
ユーザーのタスク
    ↓
PM: 要件整理 → タスク分解 → TodoWrite
    ↓
フロントエンド ＋ バックエンド（並列作業）
    ↓
セキュリティレビュー
    ↓
QA確認
    ↓
git commit & push
```

## コーディング規約

- PHP: htmlspecialchars() による出力エスケープを徹底
- CSS: クラス名はケバブケース（例: `.check-item`）
- コメントは日本語で記述
- 1ファイル1責務を維持する

## プロジェクト概要

企業のセキュリティ状況を診断し、想定被害額を算出して
セキュリティハードウェアの営業提案に活用するためのWebアプリ（PHP）。

対象ユーザー：営業担当者が訪問先企業に見せる診断ツール
